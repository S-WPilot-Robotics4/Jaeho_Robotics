# ROS2 파이썬 노드(Node) 작성 및 실행 체계 조사

## 1. rclpy.node 모듈의 Node 클래스 상속
ROS2에서 노드를 파이썬으로 작성할 때는 `rclpy.node` 모듈의 `Node` 클래스를 상속받아 사용자 정의 클래스를 만듭니다. `class LoggingNode(Node):` 와 같이 정의하며, `__init__` 생성자 내부에서 `super().__init__('노드이름')`을 호출하여 ROS2 시스템에 해당 노드의 이름을 등록합니다.

## 2. 노드 객체 생성 및 rclpy 필수 함수
클래스를 정의한 후에는 메인 함수(`main()`)에서 해당 클래스의 인스턴스(객체)를 생성하여 사용합니다. (예: `node = LoggingNode()`)
* **`rclpy.init(args=args)`:** ROS2 통신을 초기화하는 함수로, 노드를 생성하기 전에 반드시 가장 먼저 호출해야 합니다.
* **`rclpy.spin(node)`:** 노드가 종료되지 않고 계속 대기하며, 메시지 수신(콜백 함수) 등의 이벤트를 지속적으로 처리할 수 있게 해주는 무한 루프 함수입니다.
* **`rclpy.shutdown()`:** 프로그램 종료 시(Ctrl+C 등) ROS2 통신을 안전하게 차단하고 관련 리소스를 정리하는 역할을 합니다.

## 3. ROS2 노드의 로깅 시스템 (Log)
일반적인 `print()` 함수 대신 ROS2는 자체 로깅 시스템을 지원합니다. 
* **로그 기록 방법:** Node 클래스 내장 메서드인 `self.get_logger().info('메시지')`를 사용합니다. 정보의 중요도에 따라 `debug`, `info`, `warn`, `error`, `fatal` 등 5가지 수준으로 나누어 기록할 수 있습니다.
* **로그 확인 방법:** 기본적으로 노드를 실행한 터미널 창에 실시간으로 출력되며, 백그라운드에서 실행될 경우 `~/.ros/log/` 디렉토리에 텍스트 파일 형태로 저장됩니다. 또한 시각화 도구인 `ros2 run rqt_console rqt_console` 명령을 통해 발생한 모든 로그를 GUI 창에서 필터링하여 확인할 수도 있습니다.

## 4. setup.py 수정 및 빌드 (--symlink-install) 적용
새로운 파이썬 노드를 실행하기 위해서는 몇 가지 절차가 필요합니다.
1. **setup.py 수정:** `entry_points`의 `console_scripts` 항목에 `'실행할_노드명 = 패키지명.파일명:main'` 형태의 규칙을 추가하여 `ros2 run` 명령어와 파이썬 실행 코드를 연결합니다.
2. **colcon build --symlink-install:** 일반적인 빌드는 파이썬 코드를 `install` 폴더로 '복사'하지만, `--symlink-install` 옵션을 주면 '바로가기(심볼릭 링크)' 형태로 연결합니다. 즉, 빌드 이후에 파이썬 코드 내용을 수정해도 매번 다시 `colcon build`를 할 필요 없이 수정한 내용이 즉각 실행 파일에 반영되도록 해주는 매우 유용한 옵션입니다.
3. **시스템 적용:** 빌드가 완료되면 터미널에 `source install/setup.bash` 명령어를 입력해야 현재 셸 환경이 새로 빌드된 노드의 존재를 인식하고 `ros2 run`으로 실행할 수 있습니다.
