# ROS2 실습을 위한 Ubuntu 환경 구성 및 기본 명령어 학습

## 1. 우분투 리눅스 설치 환경
본 과제에서는 권장 사항에 따라 우분투 리눅스(Ubuntu 22.04 LTS)를 컴퓨터의 네이티브 운영체제로 직접 설치하여 구동하였습니다. 이를 통해 ROS2 및 하드웨어 제어 시 발생할 수 있는 가상머신 환경의 오버헤드와 호환성 문제를 사전에 차단하였습니다.

## 2. 리눅스 기본 운영체제 사용법 조사
* **터미널 실행:** 단축키 `Ctrl` + `Alt` + `T`를 사용하여 셸(Bash) 환경을 엽니다.
* **파일/디렉토리 관리 명령어:** * `ls` (목록 확인), `cd` (경로 이동), `mkdir` (폴더 생성)
  * `cp` (복사), `mv` (이동 및 이름 변경), `rm` (삭제)
* **파일 내용 확인 및 편집:** * 확인: `cat 파일명` (전체 출력), `less 파일명` (페이지 단위 출력)
  * 편집: 터미널 기반 편집기인 `nano`, `vim`이나 GUI 기반의 `code (VS Code)`를 사용합니다.
* **파일 권한 (Permission):** * 읽기(r), 쓰기(w), 실행(x) 권한으로 나뉘며 소유자, 그룹, 기타 사용자로 구분됩니다.
  * `chmod` 명령어로 권한을 변경하고, `chown` 명령어로 소유자를 변경합니다.
* **실행파일, 셸 스크립트 및 source 명령어:** * 셸 스크립트는 여러 명령어를 모아둔 실행 파일입니다.
  * `source` 명령어는 스크립트 파일의 환경 변수나 설정을 현재 셸에 즉시 적용할 때 사용합니다. (예: `source /opt/ros/humble/setup.bash`)
* **슈퍼유저와 sudo 명령어:** * 시스템 최상위 권한을 가진 관리자(root)를 의미하며, 일반 사용자가 시스템 설정 변경이나 최상위 디렉토리 접근 시 명령어 앞에 `sudo`를 붙여 임시로 최고 관리자 권한을 획득합니다.
* **패키지 관리 (apt):** * 데비안 계열 리눅스의 패키지 관리 도구입니다. `sudo apt update`로 목록을 갱신하고 `sudo apt install 패키지명`으로 소프트웨어를 설치합니다.
* **홈 디렉토리:** * 각 사용자에게 할당된 개인 기본 공간으로, `~` 기호로 축약하여 표현합니다. (예: `/home/username/` = `~`)

## 3. 실습 진행: 크롬, VS Code 설치 및 파이썬 프로그램 실행
* **설치 진행:** `apt` 패키지 매니저 및 `snap`을 활용하여 Chrome 웹 브라우저와 Visual Studio Code를 성공적으로 설치하였습니다.
* **파이썬 프로그램 실행 및 예외 처리 방법 제시:**
  * 최상위 디렉토리인 `/test`에 폴더와 파일을 생성하려면 일반 사용자 권한으로는 `PermissionDenied` 에러가 발생합니다.
  * **해결 방법:** 이를 예외 없이 실행하기 위해 임시 관리자 권한을 부여하는 `sudo` 명령어를 사용하여 스크립트를 실행했습니다.
  * `sudo python3 ~/Robotics_2026/2/1/1_hello.py`
* **실행 결과 확인 방법 제시:**
  * 파일이 정상적으로 만들어졌는지 확인하기 위해 파일 내용을 터미널에 출력하는 `cat` 명령어를 사용했습니다.
  * `cat /test/hello.txt` 명령어를 입력하여 터미널에 `Hello Linux` 문자열이 출력되는 것을 확인했습니다.
